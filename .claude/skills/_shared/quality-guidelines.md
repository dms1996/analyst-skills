# Quality Guidelines for All Skills

These guidelines apply to every skill in this collection. Reference this file in your analysis.

## 1. Pyramid Principle (Barbara Minto)

Structure all output as:
1. **Answer/Recommendation first** — Lead with the conclusion
2. **Key supporting arguments** (2-4, MECE)
3. **Evidence and data** backing each argument

Never bury the conclusion at the end. Executives read the first paragraph — make it count.

## 2. MECE (Mutually Exclusive, Collectively Exhaustive)

All categorizations, segmentations, and issue breakdowns must be:
- **Mutually Exclusive** — No overlapping categories
- **Collectively Exhaustive** — No gaps; all possibilities covered

Example: Revenue decline = (fewer customers) + (lower spend per customer). No overlap, covers all cases.

## 3. The "So What" Test

Every data point, chart, and finding must answer: **"So what does this mean for the business?"**

Chain: Data → Insight → Implication → Recommendation

Bad: "Revenue grew 10% YoY"
Good: "Revenue grew 10% YoY, outpacing the 6% industry average, suggesting market share gains. Recommend increasing investment in the channels driving this outperformance."

## 4. Action Titles

Use insight-driven titles, not descriptive labels:
- Bad: "Revenue Overview"
- Good: "Revenue accelerating at 2x industry rate, driven by enterprise segment"

## 5. Audience Adaptation

Tailor depth and language to the audience:
- **C-Suite/Board**: 1-page summary, key numbers, strategic implications, decisions needed
- **Management**: 3-5 pages, analysis with context, options and recommendations
- **Technical/Operational**: Full detail, methodology, data sources, assumptions

## 6. Professional Standards

- Use consistent terminology throughout (define in glossary if needed)
- Cite data sources for every claim
- Distinguish clearly between facts, estimates, and opinions
- Use precise language: "95% of transactions complete within 2 seconds" not "the system is fast"
- Include version/date on all deliverables
- State assumptions explicitly

## 7. Visual Communication

- Choose chart types that match the data relationship (comparison → bar, trend → line, composition → stacked bar)
- Label directly on charts rather than separate legends
- Use insight-driven chart titles
- Maintain consistent formatting across all tables and sections
- Use Markdown tables for structured data presentation

## 8. Completeness Checklist

Before finalizing any output, verify:
- [ ] Conclusion/recommendation is stated upfront
- [ ] All categories are MECE
- [ ] Every insight has a "so what" and action item
- [ ] Data sources are cited
- [ ] Assumptions are stated
- [ ] Limitations are acknowledged
- [ ] Output is appropriate for the target audience

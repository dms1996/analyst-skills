# Standard Output Formats

Consistent templates used across all skills. Reference the appropriate format in your output.

## 1. Executive Summary Format

```
## Executive Summary

**Bottom Line**: [1-2 sentence conclusion/recommendation]

**Key Findings**:
1. [Finding with quantified impact]
2. [Finding with quantified impact]
3. [Finding with quantified impact]

**Recommended Actions**:
| Priority | Action | Owner | Timeline | Expected Impact |
|----------|--------|-------|----------|-----------------|
| P0 | ... | ... | ... | ... |
| P1 | ... | ... | ... | ... |

**Risks**: [Key risks in 1-2 sentences]
```

## 2. Analysis Report Format

```
## [Insight-Driven Title]

### Executive Summary
[See format above]

### Context
[Background, scope, methodology — 2-3 paragraphs max]

### Analysis
[Main body organized in MECE sections with tables, charts descriptions, and evidence]

### Findings
| # | Finding | Impact | Confidence |
|---|---------|--------|------------|
| 1 | ... | ... | High/Medium/Low |

### Recommendations
| Priority | Recommendation | Rationale | Effort | Impact |
|----------|---------------|-----------|--------|--------|
| P0 | ... | ... | ... | ... |

### Assumptions & Limitations
- [Assumption 1]
- [Limitation 1]

### Sources
- [Source 1]
- [Source 2]
```

## 3. Comparison Table Format

```
## [Comparison Title]

### Summary
**Recommendation**: [Clear winner with reasoning]

### Detailed Comparison

| Criterion (Weight) | Option A | Option B | Option C |
|--------------------|----------|----------|----------|
| [Criterion 1] (30%) | Score/Detail | Score/Detail | Score/Detail |
| [Criterion 2] (25%) | Score/Detail | Score/Detail | Score/Detail |
| **Weighted Total** | **X.X** | **X.X** | **X.X** |

### Pros & Cons

#### Option A
- Pros: ...
- Cons: ...
- Best for: [scenario]
```

## 5. Framework Output Format (SWOT, PESTEL, Porter, etc.)

```
## [Framework Name]: [Subject]

### Summary
**Key Takeaway**: [1-2 sentence insight]

### [Framework-Specific Sections]
[Structured per framework requirements]

### Strategic Implications
| Implication | Priority | Recommended Action |
|-------------|----------|--------------------|
| ... | ... | ... |

### Confidence & Limitations
- **Confidence Level**: [High/Medium/Low]
- **Key Assumptions**: [List]
- **Data Gaps**: [What additional information would improve this analysis]
```

## 6. Process Map Format (Mermaid)

```
## [Process Name]: [AS-IS / TO-BE]

### Process Overview
[1-2 sentence description]

### Process Flow
[Mermaid diagram]

### Key Steps
| Step | Owner | Duration | Pain Points | Improvement Opportunities |
|------|-------|----------|-------------|--------------------------|

### Recommendations
[Specific improvements with expected impact]
```

## 7. KPI / Metrics Format

```
## [Metric Name]

| Attribute | Detail |
|-----------|--------|
| **Definition** | [Precise formula] |
| **Formula** | `metric = numerator / denominator` |
| **Data Source** | [Where the data comes from] |
| **Frequency** | [How often measured] |
| **Owner** | [Who is responsible] |
| **Target** | [Goal value] |
| **Benchmark** | [Industry/historical comparison] |
| **Segmentation** | [How to break down: by region, product, etc.] |
```

## 8. Confidence Badges

Use inline badges to indicate data reliability:
- **[VERIFIED]** — From official/primary sources
- **[ESTIMATED]** — Reasonable approximation
- **[ASSUMED]** — Hypothesis that needs validation
- **[ILLUSTRATIVE]** — Example data for demonstration
